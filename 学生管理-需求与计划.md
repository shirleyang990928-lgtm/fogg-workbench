> 2026-06-11 与 Shirley 确认。这是学生管理功能的总规划,每期做完在这里打勾。

## 一、需求总览(Shirley 的原始需求,翻译成功能)

| # | 需求 | 功能 | 期数 |
|---|------|------|------|
| 1 | 学生的基本信息资料 | 学生档案 | 一期 |
| 2 | 记录每堂课的上课状况 | 课堂点名:出勤+表现 | 二期 |
| 3 | 那周作业有没有做 | 作业提交跟踪 | 三期 |
| 4 | 老师有没有改 | 作业批改跟踪 | 三期 |
| 5 | 联动作业率、每周管理 | 自动统计 + 周看板 | 四期 |
| 6 | 学生面板太丑 | 学生页改版(列表+详情) | 一期 |

## 二、已拍板的决定(2026-06-11)

1. **学生档案字段**:姓名、年级/级别、**学校、城市**、家长称呼、家长联系方式、入学日期、状态(在读/停课/结课)、备注。
2. **出勤两版可切换**:简单版 = 到/缺席;专业版 = 到/迟到/缺席/请假。点名界面里一键切换,按各老师习惯。
3. **作业打分不强制**:批改 = 勾选;打分 = 选填的备注框,填了就显示,不填不影响"已批改"。

## 三、数据骨架

```
学生档案 studentsData(独立存储,user_data 表新增 students 列)
 └─ {id, name, grade, school, city, parentName, parentContact,
     enrollDate, status(在读/停课/结课), note}

课程 ←→ 学生:按"名字"关联(课程里填的学生名 = 档案里的姓名,要一致)

每节课记录(二期起,挂在课程的 classRecords 里,按日期)
 └─ 每个学生:出勤 + 表现一句话
作业(三期起,挂在每节课)
 └─ 作业内容 + 每个学生:未交→已交→已批改(+选填分数/评语)
统计(四期):出勤率、作业率、周看板、预警 —— 全自动算
```

## 四、分期计划与进度

- [x] **一期**:学生独立建档(user_data 加 students 列)+ 学生页改版为"左列表 + 右详情"可编辑档案 —— v20260611a
- [ ] **二期**:课程详情弹窗加"本节课点名"(出勤两版切换 + 表现一句话),学生详情页出现课堂时间线

## 五、技术备忘

- 学生数据存在 Supabase `user_data` 表新增的 `students` 列(jsonb),RLS 自动沿用,每位老师只能看自己的学生;
- 执行过的 SQL:`alter table public.user_data add column if not exists students jsonb;`
- 数据结构按"以后可平滑迁移到独立 students 表"设计(每个学生有稳定 id);
- 课程↔学生目前按姓名关联:**课程里填的学生名要和档案姓名一字不差**(界面里有提示)。
