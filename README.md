# Shirley Reply Sync - Codex-ready Project

这个文件夹是从你原本的单文件 HTML 拆出来的，方便用 Codex / Cursor 学 vibe coding。

## 文件说明

- `index.html`：页面骨架。放按钮、卡片、抽屉、主体区。
- `styles.css`：视觉样式。颜色、背景、卡片、按钮、布局都在这里。
- `app.js`：功能逻辑。分类切换、渲染卡片、复制、编辑话术、日程管理、导入导出都在这里。
- `original-backup.html`：你的原始完整备份。不要删，改坏时可以回退。

## 你使用 Codex 的第一条指令

请先分析这个项目，不要修改任何文件。
请告诉我：
1. index.html、styles.css、app.js 各自负责什么；
2. 现在最难维护的地方在哪里；
3. 如果我要先优化视觉，应该改哪些 CSS 区块；
4. 如果我要先优化功能，应该改哪些 JS 函数；
5. 不要重写项目，不要删除任何内容。

## 你使用 Codex 的安全规则

每次只做一种修改：

- 只改视觉：只动 `styles.css`。
- 只改功能：主要动 `app.js`，不要改视觉。
- 只改页面结构：主要动 `index.html`，不要改内容数据。
- 只整理代码：不改变页面效果、不改变功能。

## 推荐第一轮任务

请只做视觉整理，不要改变功能：
1. 保留原本温暖可爱的风格；
2. 降低背景干扰，让内容更清楚；
3. 统一按钮大小、圆角和间距；
4. 让卡片层级更明显；
5. 不删除任何现有场景、话术、分类和日程功能。

完成后请列出你修改了哪些文件，以及我需要测试哪些地方。
